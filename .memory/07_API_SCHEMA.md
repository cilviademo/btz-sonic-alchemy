# API Schema

## Scope
- This project is a JUCE plugin (C++), not a traditional HTTP API service.

## Plugin Interface Surface
- Entry point:
  - `createPluginFilter()` in `BTZ/Source/PluginProcessor.cpp`
- Processor class:
  - `BTZAudioProcessor`
- Editor class:
  - `BTZAudioProcessorEditor`

## Parameter Schema (Current)
- Source: `BTZ/Source/PluginProcessor.h` (`AudioProcessorValueTreeState`)
- Current parameter IDs (observed):
  - `punch`
  - `warmth`
  - `boom`
  - `mix`
  - `texture`
  - `drive`

## Contract Notes
- Hosts (FL Studio, etc.) interact via VST3 interface generated by JUCE.
- Backward compatibility concern:
  - Changing parameter IDs can break preset/session recall.
